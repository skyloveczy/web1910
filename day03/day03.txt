1. HTTPS和SSH
  推送时采用不同的身份认证的方式
  HTTPS每次都需要输入用户名和密码
  而SSH不需要输入用户名和密码

  使用SSH方式需要配置密钥:
    查看本机有没有密钥:
      ls -a ~/.ssh

    生成ssh密钥:
      ssh-keygen -t rsa -C "邮箱地址"

    生成2个文件:
      id_rsa : 私钥,保存在本机
      id_rsa.pub : 公钥,将其内容配置到github上

    打开公钥,复制里面的内容
      右上角头像-->settings-->SSH And GPGKeys
      -->右上角Add SSH
      -->Title 填写配置名称(随便写) content中粘贴复制的内容



Node入门
  1. 是一个Javascript的运行环境,可以让js脱离浏览器运行
  2. 是一个专注于web服务器高性能优化的专家
  3. 基于V8引擎诞生
  4. 是一个让js代码运行在服务器的开发平台
  5. 用最小的硬件成本,追求更高的开发,更高处理性能

NodeJS的特点:
  1. 单线程
  2. 非I/O阻塞
  3. 事件机制,事件环

NodeJS适合做什么
  擅长I/O,不善于计算,最擅长的是任务的调度
    InputStream,OutputStream
  收集用户表单数据
  考试系统
  群聊
  直播间



使用NodeJS运行Javascript
  1) 新建一个01.js文件
  2) 在js中输入代码
  3) 运行
    打开命令窗口,切换到js所在路径,输入命令:
    node 01.js


使用NodeJS写一个简单的服务器
  1) 引入http
    var http = require("http");
  2) 创建服务
    var server = http.createServer(function(req,res){});
  3) 监听端口,开启服务
    server.listen(4000);
  4) 在第二步的回调中处理请求并返回响应

  启动服务:
    在命令行中输入node xxx.js
    光标在闪动则说明启动成功
  
  结束程序:
    ctrl + C
